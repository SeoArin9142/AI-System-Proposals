# AI System Proposals

사내 AI 개발지원 시스템 구축안을 정리한 문서 저장소입니다.  
현재 비교 대상은 아래 두 가지입니다.

- **폐쇄망 내부형 AI 구축**
- **외부망 연계형 AI 구축**

둘 다 Owner / Reviewer / Orchestrator 구조를 만들 수 있지만, 보안 조건, 운영 부담, 비용 구조가 다릅니다.

## 목적

이 저장소는 아래 질문에 답하기 위한 문서를 제공합니다.

- 소스 외부 유출 없이 AI 개발지원 시스템을 만들 수 있는가
- 지금 이곳과 비슷한 Owner / Reviewer 구조를 어느 수준까지 재현할 수 있는가
- 폐쇄망과 외부망 중 어떤 방식이 현재 조직 조건에 더 맞는가

## 구축안 요약

### 1. 폐쇄망 내부형

권장 상황:
- 방산/폐쇄망 환경
- 소스 외부 반출 금지
- 내부 Git/Jenkins 재사용 가능
- 내부망에서만 코드/로그/문서 처리 필요

핵심 구조:
- 내부 LLM 서버
- 내부 Git / Jenkins
- Owner AI / Reviewer AI / Orchestrator
- 팀별 또는 개인별 workspace

운영 전제:
- 완전 폐쇄망이면 Claude / GPT / Gemini 최고급 상용 모델을 그대로 쓰기 어렵습니다.
- 현실적인 출발점은 **오픈소스 로컬 모델 + RAG + 근거 기반 리뷰 흐름**입니다.

첫 번째 성공 사례 권장:
- Sparrow / MISRA / 6016F 기반 Reviewer
- 입력: 정적분석 결과
- 출력: 규정 조항, 원인 설명, 수정 가이드, 패치 초안

### 2. 외부망 연계형

권장 상황:
- 외부 AI 서비스 사용이 정책상 허용됨
- 소스 일부 또는 작업 맥락을 외부로 보내는 것이 가능함
- 빠른 도입과 높은 모델 품질이 중요함

핵심 구조:
- 내부 Git / Jenkins
- 외부 AI 서비스(Claude / GPT / Gemini 등)
- 내부 오케스트레이터
- PR / 리뷰 / 테스트 흐름 연동

운영 전제:
- 최신 상용 모델 품질을 활용하기 쉽습니다.
- 대신 외부 전송 범위, 사용료, 서비스 의존성, 감사 정책을 관리해야 합니다.

## 폐쇄망 vs 외부망 비교

| 항목 | 폐쇄망 내부형 | 외부망 연계형 |
|---|---|---|
| 보안/통제 | 매우 강함 | 외부 전송 범위 통제가 필요 |
| 방산/폐쇄망 적합성 | 높음 | 낮음 또는 별도 승인 필요 |
| 초기 구축비 | 큼 | 중간 |
| 월간 운영비 | 상대적으로 예측 가능 | 인프라 + AI 사용료가 함께 누적 |
| 모델 품질 | 상용 최고급 대비 낮을 수 있음 | 최신 상용 모델 활용 가능 |
| 운영 부담 | 내부가 직접 담당 | 상대적으로 낮음 |
| 장애 영향 통제 | 내부 통제 가능 | 외부 서비스 상태 영향 받음 |
| Owner/Reviewer 시스템화 | 가능 | 가능 |
| "이곳과 같은 체감" 재현 | 구조는 가능, 모델 품질은 격차 있음 | 가장 근접 가능 |

## 장점 / 단점

### 폐쇄망 내부형 장점
- 소스 외부 유출 통제에 유리
- 방산/폐쇄망 환경에 적합
- 로그, 프롬프트, 산출물 위치를 내부에서 통제 가능
- 장기적으로 사내 자산화에 유리

### 폐쇄망 내부형 단점
- GPU 서버, 스토리지, 운영 체계가 필요
- 초기 도입비가 큼
- 모델 품질이 상용 최고급 대비 낮을 수 있음
- 모델/드라이버/운영 문제를 내부에서 해결해야 함

### 외부망 연계형 장점
- 도입 속도가 빠름
- 최신 상용 모델 품질을 즉시 활용 가능
- 초기 인프라 부담이 낮음
- 현재 상용 AI 협업 구조와 가장 유사하게 설계 가능

### 외부망 연계형 단점
- 외부 전송 허용 범위 검토 필요
- 사용량 증가 시 월 비용이 빠르게 커질 수 있음
- 서비스 정책 변화/장애 영향이 있음
- 방산/소스 외부 유출 금지 조건과 충돌 가능

## 현재 기준 권장안

### 폐쇄망이 맞는 경우
- 소스 외부 유출 금지
- 방산 또는 내부망 전용 환경
- 장기적으로 사내 전용 AI 체계를 자산화하고 싶음

### 외부망이 맞는 경우
- 빠른 도입과 높은 모델 품질이 더 중요함
- 외부 AI 사용이 허용됨
- Owner / Reviewer 시스템을 빠르게 실험해 보고 싶음

## 비용 방향

요약만 적으면:

- **폐쇄망 내부형**: 초기비용이 큼, 장기 운영 통제에 유리
- **외부망 연계형**: 초기비용은 낮거나 중간, 대신 사용료가 누적됨

정확한 수치는 아래 상세 문서를 기준으로 봅니다.
- [폐쇄망 비용/장비 상세](docs/internal-closed-network-ai-system.md)
- [외부망 비용/서비스 상세](docs/external-connected-ai-system.md)

## 결론

정리하면:

- **보안/통제 우선**이면 폐쇄망 내부형
- **속도/모델 품질 우선**이면 외부망 연계형
- **방산/소스 외부 유출 금지 조건**이면 폐쇄망이 기본 선택입니다.

다만 완전 폐쇄망에서는 현재 상용 최고급 AI와 **동일한 모델 품질**을 그대로 재현하기는 어렵습니다.  
따라서 현실적인 목표는 **오픈소스 로컬 모델 + RAG + Reviewer 우선 MVP**로 시작하는 것입니다.
